`it-article-writer` 스킬을 사용해 디지털데일리 문체로 기사를 작성하고 검증한다. 스킬의 워크플로우(SKILL.md)를 단일 기준으로 그대로 따른다.

진행 순서(반드시 준수):
1. **자료 수집(읽기만)** — 아래 `취재 자료`를 읽고 핵심만 짧게 요약·확인한 뒤, 추가 자료가 더 있는지 묻는다. 자료는 여러 차례 나뉘어 들어올 수 있으므로 **이 단계에서 기사를 쓰거나 작성 조건을 묻지 않는다.**
2. **작성 조건 확인** — 사용자가 "기사 작성"이라고 지시하면, 곧장 쓰지 말고 **제목 / 개요(중제·기사 요소) / 분량** 등 작성 조건을 먼저 묻는다.
3. **초안 작성** — 조건이 확정되면 그 조건대로 초안을 쓰고 사용자 확인을 받는다.
4. **검증** — 확인 후 웹 교차검증·팩트체크 리포트와 최종 기사를 낸다.

`취재 자료`가 비어 있으면 기사를 지어내지 말고, 사용자에게 취재 자료(텍스트 붙여넣기 / 파일 첨부 / URL)를 요청한다.
